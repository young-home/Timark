## 4. 数据库配置

### PostgreSQL 容器

**启动命令**：
```bash
docker run --name timark-postgres \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres123 \
  -e POSTGRES_DB=timark_todo \
  -p 5432:5432 \
  -d postgres:14
```

**容器管理**：
```bash
# 查看容器状态
docker ps

# 停止容器
docker stop timark-postgres

# 启动容器
docker start timark-postgres

# 重启容器
docker restart timark-postgres

# 删除容器（数据会丢失！）
docker rm -f timark-postgres
```

### 数据库连接信息

| 参数 | 值 |
|------|-----|
| **Host** | localhost |
| **Port** | 5432 |
| **Database** | timark_todo |
| **User** | postgres |
| **Password** | postgres123 |

### 连接字符串

```
postgresql://postgres:postgres123@localhost:5432/timark_todo
```

### 数据库表结构

**users 表** - 用户信息
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| email | VARCHAR(255) | 邮箱（唯一） |
| password_hash | VARCHAR(255) | bcrypt 加密密码 |
| display_name | VARCHAR(100) | 显示名称 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**todos 表** - 待办事项
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| user_id | UUID | 外键（关联 users） |
| text | TEXT | 待办内容 |
| completed | BOOLEAN | 是否完成 |
| priority | VARCHAR(20) | 优先级（high/medium/low） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 数据库操作

```bash
# 进入数据库命令行
docker exec -it timark-postgres psql -U postgres -d timark_todo

# 查看表
\dt

# 查看用户
SELECT * FROM users;

# 查看待办
SELECT * FROM todos;

# 退出
\q
```

### API 接口

#### 认证接口

**POST /api/v1/auth/register** - 用户注册
```bash
curl -X POST http://localhost:3001/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "123456",
    "displayName": "测试用户"
  }'
```

**响应**：
```json
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": "uuid",
      "email": "test@example.com",
      "displayName": "测试用户"
    }
  }
}
```

**POST /api/v1/auth/login** - 用户登录
```bash
curl -X POST http://localhost:3001/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "123456"
  }'
```

#### 待办接口

所有待办接口需要在 Header 中携带 Token：
```
Authorization: Bearer <token>
```

**GET /api/v1/todos** - 获取待办列表
```bash
curl http://localhost:3001/api/v1/todos \
  -H "Authorization: Bearer <token>"
```

**POST /api/v1/todos** - 创建待办
```bash
curl -X POST http://localhost:3001/api/v1/todos \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "新任务",
    "priority": "high"
  }'
```

**PUT /api/v1/todos/:id** - 更新待办
```bash
curl -X PUT http://localhost:3001/api/v1/todos/1 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "completed": true
  }'
```

**DELETE /api/v1/todos/:id** - 删除待办
```bash
curl -X DELETE http://localhost:3001/api/v1/todos/1 \
  -H "Authorization: Bearer <token>"
```

## 8. 故障排查

### Q1: Docker 无法启动

**症状**：Docker Desktop 一直显示 "Starting the Docker Engine..."

**解决方案**：
```powershell
# 停止所有 Docker 进程
Stop-Process -Name "Docker Desktop","com.docker.*","docker-*" -Force

# 重启 WSL
wsl --shutdown

# 重新启动 Docker Desktop
Start-Process "C:\Program Files\Docker\Docker\Docker Desktop.exe"
```

---

### Q2: 数据库连接失败

**症状**：后端启动时报数据库连接错误

**解决方案**：
```bash
# 检查容器是否运行
docker ps

# 如果没有运行，启动容器
docker start timark-postgres

# 测试连接
docker exec -it timark-postgres psql -U postgres -d timark_todo -c "SELECT 1"
```

---

### Q3: 端口被占用

**症状**：启动时提示端口已被占用

**解决方案**：
```bash
# Windows - 查找占用端口的进程
netstat -ano | findstr :5173
netstat -ano | findstr :3001
netstat -ano | findstr :5432

# 终止进程（替换 PID）
taskkill /PID <进程 ID> /F
```

---

### Q4: 后端启动失败（ES Module 错误）

**症状**：`require is not defined in ES module scope`

**解决方案**：
确保后端代码使用 ES Module 语法：
```javascript
// ✅ 正确
import express from 'express';
import { pool } from './db.js';

// ❌ 错误
const express = require('express');
```

---

### Q5: JWT token 无效

**症状**：API 返回 "Unauthorized: token 无效或已过期"

**解决方案**：
1. 检查 token 是否正确存储：`localStorage.getItem('auth_token')`
2. 重新登录获取新 token
3. 检查 `.env` 中的 `JWT_SECRET` 是否一致

---

### Q6: 前端无法连接后端

**症状**：前端调用 API 时返回网络错误

**解决方案**：
1. 确保后端服务已启动：http://localhost:3001/health
2. 检查 CORS 配置（server/index.js 中已启用）
3. 检查 API 地址配置（src/api/index.ts）
