## 2. 基础设施

### 2.1 Node.js

| 项目 | 说明 |
|------|------|
| **版本** | v18+ (当前安装：v24.12.0) |
| **用途** | 运行 JavaScript 代码的运行时环境 |
| **在本项目** | 前端构建工具 + 后端 API 服务 |

**为什么选 Node.js？**
- 前后端统一使用 JavaScript/TypeScript
- 生态丰富，npm 包齐全
- 异步 I/O 适合 API 服务
- 冷启动快，适合本地开发

**核心命令**：
```bash
node --version          # 查看版本
npm install             # 安装依赖
npm run dev             # 启动开发服务
```

| 项目 | 说明 |
|------|------|
| **版本** | 最新版 (待安装) |
| **用途** | 容器化平台 |
| **在本项目** | 运行 PostgreSQL 数据库 |

**为什么用 Docker？**


**核心命令**：
```bash

docker run <image>            # 运行容器
docker stop <container>       # 停止容器
docker rm <container>         # 删除容器
```

**代码对比**：

```typescript
// TypeScript - 有类型检查
interface Todo {
  id: number;
  text: string;
  completed: boolean;
}

function addTodo(text: string): Todo {
  return {
    id: Date.now(),
    text,        // 如果 text 是 number，编译报错
    completed: false
  };
}

// JavaScript - 无类型检查
function addTodo(text) {
  return {
    id: Date.now(),
    text,        // text 是什么类型？不知道

  };
}
```

## 4. 后端技术栈

### 4.1 Express

| 项目 | 说明 |
|------|------|
| **版本** | 4.x |
| **用途** | Node.js Web 框架 |
| **在本项目** | 本地 API 服务 |

**为什么选 Express？**

| 优势 | 说明 |
|------|------|
| **简单易学** | API 设计直观，上手快 |
| **生态丰富** | 中间件、插件数不胜数 |
| **文档完善** | 任何问题都能找到答案 |
| **社区庞大** | Stack Overflow 上有海量问答 |
| **稳定成熟** | 2010 年发布，经过时间检验 |

**核心代码**：
```javascript
const express = require('express');
const app = express();

app.use(express.json());

// 路由
app.get('/todos', async (req, res) => {
  const todos = await db.query('SELECT * FROM todos');
  res.json({ success: true, data: todos.rows });
});

app.post('/todos', async (req, res) => {
  const { text, priority } = req.body;
  const todo = await db.query(
    'INSERT INTO todos (text, priority) VALUES ($1, $2)',
    [text, priority]
  );
  res.status(201).json({ success: true, data: todo.rows[0] });
});

app.listen(3001);
```

**Express vs Fastify**：

| 特性 | Express | Fastify |
|------|---------|---------|
| 学习曲线 | 低 | 中 |
| 性能 | 基准 | 2-3 倍快 |
| 生态 | 极大 | 增长中 |
| TypeScript | 一般 | 优秀 |
| **MVP 推荐** | ✅ | - |

| **用途** | 密码加密 |

**为什么选 bcryptjs？**

**使用示例**：

```javascript
const bcrypt = require('bcryptjs');

// 加密密码
const passwordHash = await bcrypt.hash('myPassword', 10);

// 验证密码
const isValid = await bcrypt.compare('myPassword', passwordHash);
```

---

#### jsonwebtoken (JWT)

| 项目 | 说明 |
|------|------|
| **用途** | 生成和验证 JWT Token |
| **在本项目** | 用户认证 |

**为什么用 JWT？**
- 无状态认证（服务器不存 session）
- 跨域友好
- 移动端友好

**使用示例**：
```javascript
const jwt = require('jsonwebtoken');

// 生成 Token
const token = jwt.sign(
  { userId: user.id, email: user.email },
  'SECRET_KEY',
  { expiresIn: '7d' }
);

// 验证 Token
const payload = jwt.verify(token, 'SECRET_KEY');
// payload.userId, payload.email
```

| 项目 | 说明 |
|------|------|
| **用途** | 跨域资源共享中间件 |

## 5. 数据库

### 5.1 PostgreSQL

| 项目 | 说明 |
|------|------|
| **版本** | 14 |
| **用途** | 关系型数据库 |
| **在本项目** | 存储用户、待办数据 |

**为什么选 PostgreSQL？**

| 优势 | 说明 |
|------|------|
| **JSONB 支持** | 可以存储和查询 JSON 数据 |
| **高并发** | MVCC 机制，写性能优秀 |
| **复杂查询** | 窗口函数、CTE 支持完善 |
| **扩展性强** | 支持自定义函数、类型、索引 |
| **开源免费** | 完全开源，无许可费用 |

**PostgreSQL vs MySQL**：

| 特性 | PostgreSQL | MySQL |
|------|------------|-------|
| JSON 支持 | JSONB（可索引） | JSON（基础） |
| 并发写 | MVCC（更好） | 行锁 |
| 复杂查询 | 强 | 一般 |
| 学习成本 | 中 | 低 |
| **推荐** | ✅ | - |

**核心 SQL**：
```sql
-- 创建用户表
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 创建待办表
CREATE TABLE todos (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  text TEXT NOT NULL,
  completed BOOLEAN DEFAULT FALSE,
  priority VARCHAR(20) DEFAULT 'medium',
  created_at TIMESTAMP DEFAULT NOW()
);

-- 查询
SELECT * FROM todos 
WHERE user_id = 'xxx' 
  AND completed = false 
ORDER BY created_at DESC;
```

## 9. 为什么这样选型？

### 核心原则

| 原则 | 说明 |
|------|------|
| **快速上线** | 选择最成熟、最容易上手的技术 |
| **技术债可控** | 不过度设计，保持简单 |
| **易于扩展** | 预留扩展空间（如 TypeScript） |
| **团队友好** | 选择大多数人熟悉的技术 |

### 一句话总结

> **用最合适的技术，而不是最新的技术。**

每个选择都经过权衡，目标是：**快速交付 MVP，同时保持技术栈的现代性和可扩展性**。
